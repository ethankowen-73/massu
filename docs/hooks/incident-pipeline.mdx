# incident-pipeline

The `incident-pipeline` hook fires when a new incident report is written to the incidents directory. It prompts Claude to derive a prevention rule from the incident, advancing the auto-learning pipeline from Step 2 to Step 3.

## Overview

| Property | Value |
|----------|-------|
| **Event** | PostToolUse |
| **Matcher** | Write |
| **Source** | `packages/core/src/hooks/incident-pipeline.ts` |
| **Compiled** | `dist/hooks/incident-pipeline.js` |
| **Timeout** | < 500ms |
| **Part of** | Auto-Learning Pipeline (Step 2 of 4) |

## How It Works

1. Intercepts Write tool calls
2. Checks if the written file matches the incident directory pattern (default: `docs/incidents/*.md`)
3. If it matches, reads the incident report content
4. Outputs instructions for Claude to derive a prevention rule from the incident
5. Checks for existing rules to avoid duplicates

## Trigger Condition

The hook triggers when a file is written to the configured incident directory:

```
docs/incidents/2026-04-10-orders-null-check.md  → triggers
docs/incidents/2026-04-10-auth-bypass.md        → triggers
src/server/routers/orders.ts                     → does not trigger
```

## Output

When triggered, the hook outputs instructions like:

```
[INCIDENT-PIPELINE] New incident report detected.

File: docs/incidents/2026-04-10-orders-null-check.md

Next step: Derive a prevention rule from this incident.
Write a feedback memory file to memory/feedback_*.md with:
  - The rule itself (what must be done or avoided)
  - Why (reference to the incident)
  - How to apply (when/where the rule should be enforced)
```

## Configuration

Controlled via `autoLearning.incidentDir` in `massu.config.yaml`:

```yaml
autoLearning:
  incidentDir: docs/incidents    # Directory to watch for incident reports
```

## Pipeline Context

```
fix-detector ──► [incident-pipeline] ──► rule-enforcement-pipeline ──► auto-learning-pipeline
                    (YOU ARE HERE)
```

This is Step 2. After the fix-detector flags a fix, an incident report is written. This hook then prompts the derivation of a prevention rule (Step 3).
