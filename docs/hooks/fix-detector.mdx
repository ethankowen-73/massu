# fix-detector

The `fix-detector` hook detects when a bug fix is being applied during a session by analyzing git diffs of edited files. When fix signals are detected, it sets session-level state so the `auto-learning-pipeline` hook can enforce pipeline completion at session end.

## Overview

| Property | Value |
|----------|-------|
| **Event** | PostToolUse |
| **Matcher** | Edit, Write |
| **Source** | `packages/core/src/hooks/fix-detector.ts` |
| **Compiled** | `dist/hooks/fix-detector.js` |
| **Timeout** | < 1000ms |
| **Part of** | Auto-Learning Pipeline (Step 1 of 4) |

## How It Works

1. Intercepts Edit and Write tool calls on tracked source files
2. Runs `git diff` on the edited file to get the actual changes
3. Applies fix detection heuristics against the diff output
4. If fix signals are detected, appends a JSON record to a session-level flag file
5. The `auto-learning-pipeline` hook reads this flag file at session end

## Fix Detection Heuristics

The hook checks diffs against these signal patterns:

| Signal | What It Detects |
|--------|----------------|
| `removed_broken_code` | Lines removed that contain "bug", "broken", "wrong", "incorrect", "typo", "crash", "error", "fail", "miss", "stale" |
| `added_error_handling` | Addition of try/catch blocks, error handling, or validation |
| `method_name_correction` | A renamed method or function |
| `auth_fix` | Changes to authentication or authorization code |
| `nil_handling_fix` | Addition of null/nil/undefined checks |
| `concurrency_fix` | Changes to locking, mutex, or race condition handling |
| `async_pattern_fix` | Corrections to async/await patterns |
| `added_missing_import` | Addition of previously missing imports |

## Configuration

Controlled via `autoLearning.fixDetection` in `massu.config.yaml`:

```yaml
autoLearning:
  fixDetection:
    enabled: true        # Set to false to disable fix detection
    lookbackDays: 7      # Days of git history to consider
    signals:             # Which signal types to detect (all enabled by default)
      - removed_broken_code
      - added_error_handling
      - method_name_correction
      - auth_fix
      - nil_handling_fix
      - concurrency_fix
      - async_pattern_fix
      - added_missing_import
```

## Output

The hook writes fix signals as JSONL to a temporary session flag file (`/tmp/massu-auto-learning/fixes-{session_id}.jsonl`). Each line contains:

```json
{
  "file": "src/server/routers/orders.ts",
  "signals": ["removed_broken_code", "added_error_handling"],
  "timestamp": "2026-04-10T14:23:01.000Z"
}
```

No output is injected into the Claude Code session -- the hook operates silently.

## Pipeline Context

```
[fix-detector] ──► incident-pipeline ──► rule-enforcement-pipeline ──► auto-learning-pipeline
   (YOU ARE HERE)
```

This is Step 1. When a fix is detected, the auto-learning-pipeline hook (Step 4) will enforce that Steps 2 and 3 are completed before the session can end.
