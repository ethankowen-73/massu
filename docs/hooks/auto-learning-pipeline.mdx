# auto-learning-pipeline

The `auto-learning-pipeline` hook runs at session end and checks whether bug fixes detected during the session have been fully processed through the incident → rule → enforcement pipeline. If the pipeline is incomplete, it outputs mandatory instructions for Claude to follow before ending the session.

## Overview

| Property | Value |
|----------|-------|
| **Event** | Stop |
| **Matcher** | (all) |
| **Source** | `packages/core/src/hooks/auto-learning-pipeline.ts` |
| **Compiled** | `dist/hooks/auto-learning-pipeline.js` |
| **Timeout** | < 2000ms |
| **Part of** | Auto-Learning Pipeline (Step 4 of 4) |

## How It Works

1. Reads the session flag file written by `fix-detector` to find all detected fixes
2. For each fix, checks whether:
   - An incident report exists in `docs/incidents/` (if `pipeline.requireIncidentReport` is true)
   - A prevention rule exists in `memory/feedback_*.md` (if `pipeline.requirePreventionRule` is true)
   - An enforcement hook or config entry was placed (if `pipeline.requireEnforcement` is true)
3. If any pipeline steps are missing, outputs instructions for Claude to complete them
4. Cleans up the session flag file after processing

## Output

When the pipeline is incomplete, the hook outputs structured instructions:

```
[AUTO-LEARNING] Bug fixes detected but pipeline incomplete.

Fixes detected:
  - src/server/routers/orders.ts (removed_broken_code, added_error_handling)

Missing steps:
  ☐ Write incident report to docs/incidents/
  ☐ Derive prevention rule in memory/feedback_*.md
  ☐ Place enforcement in project hooks or config

You MUST complete these steps before ending the session.
```

When the pipeline is complete (or no fixes were detected), the hook exits silently.

## Configuration

Controlled via `autoLearning.pipeline` in `massu.config.yaml`:

```yaml
autoLearning:
  pipeline:
    requireIncidentReport: true   # Require incident report for each fix
    requirePreventionRule: true    # Require prevention rule derivation
    requireEnforcement: true       # Require enforcement placement
```

Set any requirement to `false` to make that step optional.

## Pipeline Context

```
fix-detector ──► incident-pipeline ──► rule-enforcement-pipeline ──► [auto-learning-pipeline]
                                                                        (YOU ARE HERE)
```

This is Step 4 -- the forcing function that ensures no bug fix goes undocumented or unenforced.
