# /massu-article-review

Analyze technical articles, blog posts, and documentation to extract actionable insights for project development. Produces structured summaries with a Project Fit Score (0-100) and automatic comparison to current project approaches.

## Usage

```
/massu-article-review https://example.com/article
/massu-article-review                              # Will prompt for input
```

Accepts a URL argument, pasted article content, or prompts for input if neither is provided. If a URL is behind a paywall, the command will ask for alternative input (copy/paste, PDF, or saved HTML).

## Protocol Steps

1. **Parse input** -- check arguments for URL or pasted content; prompt if neither
2. **Fetch content** -- use WebFetch for URLs; handle paywall failures gracefully
3. **Analyze article** -- apply the full analysis framework:
   - Executive summary (2-3 sentences)
   - Key points extraction table
   - Technology/tool overview (maturity, community, prerequisites)
   - **Mandatory comparison to current project approach** (gap analysis, overlap with existing commands/patterns, what each approach does better)
4. **Calculate Project Fit Score** -- weighted across 6 components:
   - Problem Relevance (25 pts), Architecture Fit (20 pts), Pattern Compliance (20 pts), Implementation Effort (15 pts), Risk Level (10 pts), Maintenance Burden (10 pts)
5. **Score interpretation**: 80-100 ADOPT, 60-79 TRIAL, 40-59 ASSESS, 20-39 HOLD, 0-19 AVOID
6. **Generate implementation path** (if score >= 40) with phased approach
7. **Save review** to a dated markdown file in the project docs directory
8. **Silently self-score** quality checks and append to `command-scores.jsonl`

## When to Use

- When someone shares a technical article, tweet, or blog post and wants analysis of how it applies to the project
- When evaluating whether to adopt a new tool, library, or pattern
- When comparing an external approach to your existing architecture
- Before committing to a technology decision based on an article's recommendation
